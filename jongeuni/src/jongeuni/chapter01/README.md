### 회고
챕터 1에서는 절차지향 -> 객체지향으로 변경하면서 객체지향의 장점을 확실히 알아볼 수 있었다.

책에 있는 코드를 따라치는 것에 대한 필요성을 크게 못 느꼈었는데, 따라치면서 코드를 확인하니 이해도 더 잘 되고 긍정적인 부분이 있는 것 같다.

마지막에 '객체지향이 안정감을 준다'는 부분에 무척 공감이 되었고, 앞으로의 내용들이 더욱 기대가 된다.
